# CRM Avento — как запустить (Фаза 1: единый инбокс)

Коротко и по шагам. Технические детали — минимум.

## Что уже сделано (Фаза 1, MVP)
- **Коллекции CRM** в базе: Лиды, Контакты, Сообщения, События, **Каналы** (подключённые
  аккаунты-источники: 2 сайта, WhatsApp, Telegram, 3 Instagram и т.д.).
- **Приём заявок с сайта** (`/api/lead`) — теперь заявки **сохраняются** в CRM (раньше просто
  терялись), с меткой источника, UTM и рефереру.
- **Общий приёмник** `/api/intake` — сюда шлют заявки внешние сайты (защищён ключом).
- **Вебхуки Telegram и WhatsApp** — принимают входящие сообщения в инбокс.
- **Доска-инбокс** `/crm/inbox` — колонки: Новый → Начали разговор → Квалифицирован →
  Этап закрытия → Выиграно / Проиграно. На каждой карточке видно **откуда пришёл лид**.
- **PWA** — CRM можно «поставить иконкой» на iPhone.

## Что нужно от тебя, чтобы запустить (5 минут)
1. **Создать базу данных Neon** (бесплатно): https://neon.tech → New Project → скопировать
   «Connection string» (начинается с `postgres://…`).
2. Прислать мне (или вставить в файл `site/.env.local`) две строки:
   ```
   DATABASE_URL=postgres://…   ← из Neon
   PAYLOAD_SECRET=любая-длинная-случайная-строка
   ```
3. Я запущу локально, создадим первого пользователя-менеджера и проверим, что заявка с сайта
   долетает до доски.

## Запуск локально (для справки)
```
cd site
nvm use 22           # ВАЖНО: Node 22, не 24 (см. ниже)
npm install          # один раз
npm run dev -- -p 3011
```
Открыть: инбокс — http://localhost:3011/crm/inbox · админка — http://localhost:3011/admin

## Переменные окружения (по мере подключения каналов)
Обязательные сейчас:
- `DATABASE_URL` — база Neon
- `PAYLOAD_SECRET` — секрет для входа в админку

Появятся позже, по каналам:
- `INTAKE_API_KEY` — ключ для приёма заявок с внешних сайтов (`/api/intake`)
- `RESEND_API_KEY`, `LEAD_NOTIFY_TO`, `LEAD_NOTIFY_FROM` — письма-уведомления о новых лидах
- `TELEGRAM_BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, `TELEGRAM_BOT_ID` — Telegram
- `WHATSAPP_VERIFY_TOKEN`, `WHATSAPP_APP_SECRET` — WhatsApp (через агрегатора)
- `ANTHROPIC_API_KEY` — позже, для ИИ-агента (Фаза 3)

## ⚠️ Важно про Node
Генератор типов и CLI Payload **ломаются на Node 24** (и 22.x в некоторых операциях) из-за
несовместимости (`ERR_REQUIRE_ASYNC_MODULE`). Сам сайт (`npm run dev`) работает нормально.
Для генерации типов (`npm run generate:types`) и миграций используем Node 20.18 (или CI).
Это разовая настройка окружения, к коду отношения не имеет.

## Как настроить каналы (после запуска базы)
1. В админке `/admin` → **Каналы (Channels)** → создать записи:
   - «Сайт Global» (тип site, identifier = домен), «Сайт Israel», «WhatsApp +351…»,
     «Telegram @avento_bot», «Instagram @avento.porto» ×3 и т.д.
2. Каждый входящий лид автоматически привяжется к нужному каналу по его идентификатору —
   так ты всегда видишь точный источник.
